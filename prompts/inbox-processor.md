# Inbox笔记处理Skill

**功能**：自动化处理Inbox中的笔记，包括类型判断、链接推荐、质量检查

---

## 使用方法

在对话中调用：
```
请帮我处理这个Inbox笔记：[[01_Inbox/202601061634_07 JVM调优工具.md]]
```

或批量处理：
```
请处理Inbox中的所有JVM相关笔记
```

---

## 核心功能

### 1. 自动判断笔记类型

基于决策树分析笔记内容：
- **03_Notes/**：技术知识点（概念、原理、操作步骤）
- **04_Projects/**：项目实践（有时间线、具体目标、"我做了什么"）

**判断依据**：
- 文件名分析
- 前10行内容分析
- 结构特征（Q&A格式、代码示例、叙述性文字）

### 2. 推荐Related链接

搜索策略：
1. **同技术栈**：搜索相同关键词的笔记
2. **同系列**：识别JVM系列、Spring系列等
3. **关联概念**：搜索相关技术术语

推荐数量：2-4个（按相关性排序）

### 3. 自动清理frontmatter

执行操作：
```yaml
tags: []  # 清理 ['待链接']
status: 生长  # 可选：从"种子"改为"生长"
```

保持不变：
- type, ddc, created

### 4. 质量检查

检查项：
- [ ] MOC链接可点击
- [ ] 图片路径正确
- [ ] Related链接文件存在
- [ ] 外部链接有效

### 5. 生成处理报告

输出包含：
- 判断结果（03_Notes/ 或 04_Projects/）
- 推荐的Related链接
- 需要手动检查的项目
- 移动建议

---

## 处理流程

1. **读取文件**：获取笔记内容和frontmatter
2. **分析内容**：应用决策树判断类型
3. **搜索关联**：基于关键词搜索相关笔记
4. **生成建议**：推荐Related链接和目标目录
5. **执行操作**：清理frontmatter、更新Links
6. **质量检查**：验证所有链接和路径
7. **生成报告**：输出完整的处理结果

## 技术细节

**决策树逻辑**：
```
IF 包含"我做了"、"我遇到的问题" → Project
ELIF 包含"步骤"、"教程"、"使用方法" + 技术术语 → Note
ELIF 文件名包含"复盘"、"总结"、"计划" → Project
ELSE → Note（默认）
```

**关键词提取**：
- 技术栈：JVM, Spring, Vue, Redis, MySQL...
- 概念：内存, 线程, 组件, 响应式...
- 动作：部署, 配置, 优化, 调试...

**链接搜索策略**：
1. 搜索文件名中的关键词
2. 搜索内容中的关键词
3. 优先选择同DDC的笔记
4. 限制推荐数量为2-4个
