# Article Visual Style System

Use one article-level visual style for the cover and all body illustrations.

## Selection

Recommend 3 styles when the user has not specified one:

- Pick for article content, audience, and publishing channel.
- Explain each option in one short reason.
- Ask once, then reuse the selected style for all generated images.

## Core Styles

| Style | Best for |
|-------|----------|
| `elegant` | Business, thought leadership, mature analysis |
| `flat-doodle` | Productivity tools, SaaS workflows, friendly tutorials |
| `blueprint` | Architecture, systems, technical process |
| `bold-editorial` | Strong opinions, launches, public-facing essays |
| `chalkboard` | Courses, explanations, workshop notes |
| `editorial-infographic` | Journalism, explainers, article concepts with structure |
| `intuition-machine` | Technical or academic articles with labels |
| `minimal` | Philosophy, core concepts, quiet arguments |
| `notion` | Knowledge work, productivity, digital systems |
| `pixel-art` | Developer culture, retro tech, playful technical stories |
| `sketch-notes` | Learning notes, practical tutorials, knowledge sharing |
| `vector-illustration` | Educational concepts, approachable brand visuals |
| `vintage` | History, biography, tradition, archival themes |
| `warm` | Personal growth, human stories, lived experience |
| `watercolor` | Lifestyle, travel, gentle personal essays |

If a legacy sub-skill has a more specific matching style, use that style name directly.
